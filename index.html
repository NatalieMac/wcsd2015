What is responsive?

* Flexible grids + flexible images + media queries
* Not a replacement or solution for mobile
* Anyone, anywhere
* Smart phone, feature phone, gaming console, ereader, tablet, watch, smart fridge, or some other device

Responsive Workflow

* No experts
* Process I'm experimenting with

Three Phases

  • Content, prioritization, and structure
  • Develop-y design work
  • Design-y development work
* Phase 1, what the website will do

1. Identify tasks

* What tasks will users perform on the site?

2. Task-specific content

* What content will help users do their tasks?

3. Group into pages

* Group tasks that make sense together

4. Priority Structure

* Stay focused on what pages do, not what they look like
* Phase 1 complete, move on to develop-y design work
* Develop-y because much is done in code

5. Typography

* Not limited to web-safe
* Set up vertical rythm, modular scale, ideal measure
* Stick to one or choose a pairing
* Art, not science
* Most content is still text, so that's where we start

6. Color Texture

* Choose color schemes
* Design motifs

7. Element Design

* Individual elements - button, form field, headline, nav
* Take cues from type

by Dan Mall

* Omnes Pro - rounded, bubbly typeface
* Bubble gum color
* Rounded corner design

by Dan Mall

* Brothers - chiseled, angular
* Jewel tones
* Angled corners

by Dan Mall

* Bodoni
* High-contrast - thick and thin lines
* Play off contrast with headline borders

8. Module Design

* Assemble elements into modules
* Adjust any awkwardness
* Somewhere around here the design-y development work begins

9. Layouts

* Experiment with different ways of combining modules into layouts
* Experiment with ways the layouts can respond to different screen sizes
* Start with small screens and work up

10. Interactivity

* CSS and/or JS
* Move to cycle

11. Review Refine

* Tight cycle inside organization
* Bigger cycle to loop in client
* Content-out, not canvas-in
* No canvas-in design tools
* Summary - start w/ content, structure, flexibility, optimize for context
